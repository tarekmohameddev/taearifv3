## المكونات العالمية (Global Components)

يوجد في المشروع مكونان عالميان يتم استخدامهما عبر جميع الصفحات وهما Header و Footer. هذه المكونات لها نظام إدارة خاص ومختلف عن المكونات العادية:

### 1. المكونات العالمية المتاحة:
- **StaticHeader1**: `components/tenant/header/StaticHeader1.tsx`
- **StaticFooter1**: `components/tenant/footer/StaticFooter1.tsx`

### 2. الـ Structures الخاصة بالمكونات العالمية:
- **Header Structure**: `componentsStructure/header.ts` - يحتوي على تعريف كامل لحقول الـ header مع جميع الخصائص مثل:
  - Position (sticky, fixed, static)
  - Height (desktop, tablet, mobile)
  - Background (solid, gradient)
  - Colors (text, link, icon, border, accent)
  - Logo (image, text, font settings)
  - Menu items (array of navigation links)
  - Actions (search, user profile, cart)
  - Responsive settings
  - Animations

- **Footer Structure**: `componentsStructure/footer.ts` - يحتوي على تعريف كامل لحقول الـ footer مع:
  - Background (image, color, gradient, overlay)
  - Layout (columns, spacing, padding)
  - Content (company info, quick links, contact info, social media)
  - Footer bottom (copyright, legal links)
  - Styling (colors, typography, spacing, effects)

### 3. الـ Stores المرتبطة بالمكونات العالمية:

#### في `context/editorStore.ts`:
- **globalHeaderData**: `ComponentData` - بيانات الـ header العالمية
- **globalFooterData**: `ComponentData` - بيانات الـ footer العالمية
- **setGlobalHeaderData**: دالة لتحديث بيانات الـ header العالمية
- **setGlobalFooterData**: دالة لتحديث بيانات الـ footer العالمية
- **updateGlobalHeaderByPath**: دالة لتحديث حقل محدد في الـ header
- **updateGlobalFooterByPath**: دالة لتحديث حقل محدد في الـ footer

#### في `context/editorStoreFunctions/`:
- **headerFunctions.ts**: يحتوي على:
  - `getDefaultHeaderData()`: البيانات الافتراضية للـ header
  - `headerFunctions.ensureVariant()`: إنشاء variant جديد
  - `headerFunctions.getData()`: الحصول على البيانات
  - `headerFunctions.setData()`: تحديث البيانات
  - `headerFunctions.updateByPath()`: تحديث حقل محدد

- **footerFunctions.ts**: يحتوي على:
  - `getDefaultFooterData()`: البيانات الافتراضية للـ footer
  - `footerFunctions.ensureVariant()`: إنشاء variant جديد
  - `footerFunctions.getData()`: الحصول على البيانات
  - `footerFunctions.setData()`: تحديث البيانات
  - `footerFunctions.updateByPath()`: تحديث حقل محدد

### 4. إدارة المكونات العالمية في الـ Live Editor:

#### في `components/tenant/live-editor/EditorSidebar/index.tsx`:
- يتم التعامل مع المكونات العالمية بشكل خاص في دالة `handleSave()`
- عند حفظ الـ header: `selectedComponent.id === "global-header"`
- عند حفظ الـ footer: `selectedComponent.id === "global-footer"`
- يتم استخدام `setGlobalHeaderData()` و `setGlobalFooterData()` لحفظ التغييرات

#### في `components/tenant/live-editor/EditorSidebar/components/AdvancedSimpleSwitcher.tsx`:
- يتم التعامل مع تحديثات المكونات العالمية عبر:
  - `updateGlobalHeaderByPath()` للـ header
  - `updateGlobalFooterByPath()` للـ footer

#### في `components/tenant/live-editor/LiveEditorHooks.tsx`:
- يتم التعرف على المكونات العالمية في `selectedComponent`:
  - `selectedComponentId === "global-header"` للـ header
  - `selectedComponentId === "global-footer"` للـ footer

### 5. العرض النهائي للعميل:

#### في `app/tenant/[tenantId]/[slug]/page.tsx`:
- يتم استيراد المكونات العالمية: `StaticHeader1` و `StaticFooter1`
- يتم عرض الـ header في أعلى الصفحة: `<StaticHeader1 />`
- يتم عرض الـ footer في أسفل الصفحة: `<StaticFooter1 />`
- يتم تصفية مكونات الـ header والـ footer من `filteredComponentsList` لأنها مكونات عالمية

#### في `components/tenant/header/StaticHeader1.tsx`:
- يستخدم `useEditorStore((s) => s.globalHeaderData)` للحصول على البيانات
- يستخدم `useTenantStore` للحصول على بيانات الـ tenant
- يقوم بدمج البيانات الافتراضية مع البيانات العالمية
- يعرض الـ header مع جميع الإعدادات المخصصة

#### في `components/tenant/footer/StaticFooter1.tsx`:
- يستخدم `useEditorStore((s) => s.globalFooterData)` للحصول على البيانات
- يستخدم `useTenantStore` للحصول على بيانات الـ tenant
- يقوم بدمج البيانات الافتراضية مع البيانات العالمية
- يعرض الـ footer مع جميع الإعدادات المخصصة

### 6. الفروق الرئيسية بين المكونات العالمية والمكونات العادية:

1. **التخزين**: المكونات العالمية تُحفظ في `globalHeaderData` و `globalFooterData` بدلاً من `pageComponentsByPage`
2. **الاستخدام**: تُستخدم عبر جميع الصفحات ولا تحتاج لإضافتها لكل صفحة
3. **التحديث**: يتم تحديثها عبر دوال خاصة مثل `updateGlobalHeaderByPath`
4. **العرض**: تُعرض تلقائياً في كل صفحة دون الحاجة لإضافتها يدوياً
5. **التصفية**: يتم تصفيتها من قائمة مكونات الصفحة لأنها مكونات عالمية

### 7. آلية العمل:
1. عند تحميل البيانات من قاعدة البيانات، يتم تحميل `globalHeaderData` و `globalFooterData`
2. في الـ live editor، يمكن تعديل هذه المكونات عبر الـ sidebar
3. التغييرات تُحفظ في الـ store العالمي
4. في العرض النهائي، تقوم المكونات بقراءة البيانات من الـ store العالمي
5. يتم دمج البيانات الافتراضية مع البيانات المخصصة للعرض

## ربط المكونات العالمية بالـ Stores والـ Editor Sidebar

### 1. ربط المكونات العالمية بالـ Stores:

#### في `StaticHeader1.tsx`:
```typescript
// يستخدم useEditorStore للحصول على البيانات العالمية

// يستخدم useTenantStore للحصول على بيانات الـ tenant

```

#### في `StaticFooter1.tsx`:
```typescript
// نفس النمط للـ footer
const globalComponentsData = useEditorStore((s) => s.globalComponentsData);
const globalFooterData = globalComponentsData?.footer;

const tenantData = useTenantStore((s) => s.tenantData);
const tenantGlobalComponentsData = tenantData?.globalComponentsData;

```

### 2. آلية دمج البيانات (Data Merging):

#### أولوية البيانات في StaticHeader1.tsx:
```typescript
// 1. البيانات الافتراضية (أقل أولوية)
let result = { ...defaultData };

// 2. بيانات الـ editorStore (أولوية متوسطة)
if (globalHeaderData && Object.keys(globalHeaderData).length > 0) {
  result = deepMerge(result, globalHeaderData);
}

// 3. بيانات الـ tenantStore (أولوية عالية)
if (tenantGlobalHeaderData && Object.keys(tenantGlobalHeaderData).length > 0) {
  result = deepMerge(result, tenantGlobalHeaderData);
}

// 4. بيانات الـ override (أعلى أولوية)
if (overrideData && Object.keys(overrideData).length > 0) {
  result = deepMerge(result, overrideData);
}
```

#### دالة Deep Merge:
```typescript
const deepMerge = (target: any, source: any): any => {
  if (!source || typeof source !== 'object') return target || source;
  if (!target || typeof target !== 'object') return source;
  
  const result = { ...target };
  
  for (const key in source) {
    if (source.hasOwnProperty(key)) {
      if (source[key] && typeof source[key] === 'object' && !Array.isArray(source[key])) {
        result[key] = deepMerge(target[key], source[key]);
      } else {
        result[key] = source[key];
      }
    }
  }
  
  return result;
};
```

```typescript
updateByPath: (path, value) => {
  // تحليل المسار
  const segments = path.replace(/\[(\d+)\]/g, ".$1").split(".").filter(Boolean);
  
  // إنشاء البيانات الجديدة
  let newData = { ...(state.tempData || {}) };
  let cursor = newData;
  
  // التنقل في المسار
  for (let i = 0; i < segments.length - 1; i++) {
    const key = segments[i];
    const nextIsIndex = !Number.isNaN(Number(segments[i + 1]));
    
    // إنشاء الكائنات/المصفوفات المطلوبة
    if (existing == null || typeof existing === "string") {
      cursor[key] = nextIsIndex ? [] : {};
    }
    cursor = cursor[key];
  }
  
  // تعيين القيمة النهائية
  const lastKey = segments[segments.length - 1];
  cursor[lastKey] = value;
  
  return { tempData: newData };
}
```

```typescript
  addContactCard: (currentData: ComponentData, card: any): ComponentData => ({
    ...currentData,
    cards: [...(currentData.cards || []), card]
  }),
  
  removeContactCard: (currentData: ComponentData, index: number): ComponentData => ({
    ...currentData,
    cards: (currentData.cards || []).filter((_, i) => i !== index)
  }),
  
  updateContactCard: (currentData: ComponentData, index: number, updates: any): ComponentData => ({
    ...currentData,
    cards: (currentData.cards || []).map((card, i) => i === index ? { ...card, ...updates } : card)
  }),
  
  addLinkToCard: (currentData: ComponentData, cardIndex: number, link: any): ComponentData => ({
    ...currentData,
    cards: (currentData.cards || []).map((card, i) => {
      if (i === cardIndex) {
        return {
          ...card,
          content: { ...card.content, links: [...(card.content?.links || []), link] }
        };
      }
      return card;
    })
  }),
  
  validate: (data: ComponentData): { isValid: boolean; errors: string[] } => {
    const errors: string[] = [];
    
    if (!data.cards || data.cards.length === 0) {
      errors.push("At least one contact card is required");
    }
    
    if (data.cards) {
      data.cards.forEach((card: any, index: number) => {
        if (!card.icon?.src) {
          errors.push(`Card ${index + 1} is missing icon source`);
        }
        if (!card.title?.text) {
          errors.push(`Card ${index + 1} is missing title`);
        }
        // ... المزيد من التحقق
      });
    }
    
    return { isValid: errors.length === 0, errors };
  }
};
```
